# 台球智能AR投影系统 - 设计文档

## 概述

为中式八球台球桌开发一套智能AR投影系统，通过WiFi摄像头获取桌面视频，使用电脑进行视觉识别和物理计算，通过智能投影仪将进攻路线投射到桌面上，通过安卓手机APP进行控制，通过独立液晶屏显示比分和统计信息。

## 硬件架构

### 设备清单

| 设备 | 规格 | 用途 |
|------|------|------|
| 台球桌 | 中式八球标准台 | 打球桌面 |
| WiFi摄像头 | 支持RTSP协议 | 拍摄桌面，通过WiFi传输到电脑 |
| 智能投影仪 | 2000元内，安卓系统 | 投射路线到桌面，WiFi接收画面 |
| 电脑 | Windows, RX 7900 XTX, 32GB | 后台运算（另一房间，无显示器） |
| 安卓手机 | 用户现有手机 | 安装控制APP |
| NAS/软路由 | 黑群晖8100 / iStoreOS J4125 | 浏览器显示比分网页，VGA输出到液晶屏 |
| 液晶屏 | 用户自备 | 显示比分/训练统计（VGA接口） |

### 网络拓扑

```
📷 WiFi摄像头 ──┐
                │ WiFi
📽️ 智能投影仪 ──┤
                ├─── 🌐 路由器 ─── 💻 电脑 (Windows)
📱 安卓手机 ────┘                          │
                                           │ HTTP/WebSocket
                                    ┌──────┴──────┐
                                    🖥️ NAS/软路由 ── VGA ── 🖥️ 液晶屏
```

## 软件架构

### 1. 电脑后端 (Python)

#### 技术栈
- **Python 3.11+** - 主开发语言
- **OpenCV** - 视觉识别（摄像头RTSP读取、球体检测、颜色识别）
- **NumPy** - 数学计算
- **FastAPI** - Web API服务器（HTTP + WebSocket）
- **Uvicorn** - ASGI服务器
- **自研物理引擎** - 台球路线计算
- **Pillow / Cairo** - 投影画面渲染

#### 模块划分

##### A. 摄像头模块 (Camera Module)
- 通过RTSP协议连接WiFi摄像头
- 持续读取视频帧
- 自动重连机制
- 帧率控制（建议5-10fps用于检测）

##### B. 视觉识别模块 (Vision Module)
- **台球桌检测**: 使用边缘检测 + 霍夫变换找到桌子边框
- **透视矫正**: 将摄像头斜拍画面转为俯视图
- **球体检测**: 使用霍夫圆检测 + 颜色空间分割
- **球类型识别**: 
  - 区分花色球（Stripes）和纯色球（Solids）
  - 识别黑8（黑色）
  - 识别白球（母球，白色）
- **位置跟踪**: 记录每颗球的坐标
- **进袋检测**: 检测球是否进入球袋

##### C. 物理引擎 (Physics Engine)
- **碰撞计算**: 球与球之间的弹性碰撞
- **库边反弹**: 球与桌岸的碰撞（吃一库、两库等）
- **摩擦力模拟**: 球在桌面上的减速
- **旋转效应**: 高杆（跟球）、低杆（拉球）、侧旋效果
- **路线搜索**:
  - 直接进球路线
  - 翻袋路线（一库/多库）
  - 最佳角度推荐

##### D. API服务器 (API Server)
- **REST API**:
  - `GET /api/status` - 系统状态
  - `GET /api/table` - 桌面实时状态（球的位置）
  - `POST /api/mode` - 切换模式（比赛/训练）
  - `POST /api/control/start` - 启动系统
  - `POST /api/control/stop` - 停止系统
  - `GET /api/score` - 比赛比分
  - `GET /api/training/levels` - 训练关卡列表
  - `POST /api/training/select-level` - 选择训练关卡
  - `POST /api/training/verify-placement` - 验证摆球位置
- **WebSocket**:
  - `/ws/camera` - 实时摄像头画面推送
  - `/ws/projector` - 投影画面推送
  - `/ws/table-state` - 桌面状态实时更新

##### E. 投影渲染模块 (Projector Renderer)
- 根据物理引擎计算结果
- 将路线绘制为图像
- 标注：白球路径（蓝色）、目标球路径（黄色）、球袋标记
- 通过WebSocket推送给投影仪APP

##### F. 比分网页 (Scoreboard Web)
- 纯HTML+CSS+JavaScript 页面
- 自动通过WebSocket获取最新数据
- 比赛模式：双方比分、局数
- 训练模式：关卡进度、成功率
- 大字显示，远距离可读

### 2. 安卓手机APP

#### 技术栈
- **Java/Kotlin** + **Android SDK**
- **WebSocket客户端** - 连接电脑后端
- **MVP/MVC架构**

#### 界面设计

- **首页**: 自动发现后端电脑，显示连接状态
- **主界面**: 模式选择（比赛/训练/闯关）
- **比赛界面**: 
  - 显示当前桌面状态
  - 显示推荐路线预览
  - 显示比分
  - 开始/暂停/结束比赛
- **训练界面**:
  - 关卡列表（1-10档）
  - 当前题目显示（目标球位置、母球位置）
  - 摆球确认按钮
  - 路线/杆法/力度显示
  - 训练结果反馈
- **闯关模式界面**:
  - 当前关卡进度（需连续成功3次）
  - 关卡选择（已解锁的关卡）

### 3. 投影仪安卓APP

#### 技术栈
- **Java/Kotlin** + **Android SDK**
- **WebSocket客户端** - 接收投影画面
- **开机自启动** - BOOT_COMPLETED广播接收器

#### 功能
- 开机自动启动
- 全屏显示（隐藏状态栏）
- 通过WebSocket接收后端推送的投影画面
- 低延迟显示（帧缓存 + 预加载）
- 连接断开时显示重连提示

#### 开机自启动实现
- 注册 `BOOT_COMPLETED` 广播接收器
- 申请 `SYSTEM_ALERT_WINDOW` 权限
- 在投影仪设置中允许该APP自启动
- 对于无法安装APK的投影仪，可使用当贝投影、小米投影等支持U盘安装的应用市场

#### APP安装方式
- 我会提供完整的Android项目源代码和编译好的APK安装包
- 投影仪APP：通过U盘复制APK文件到投影仪上安装
- 手机APP：通过U盘或微信发送APK文件安装
- 部分投影仪需要用遥控器开启"允许安装未知来源应用"

### 4. 比分网页

- 电脑后端提供HTTP服务
- NAS/软路由用浏览器打开，全屏显示
- 自动刷新（WebSocket实时推送）
- 比赛模式：大字比分，一目了然
- 训练模式：当前关卡、成功/失败统计

## 比赛模式详细设计

### 规则
- 双方轮流击球（一方打纯色球，一方打花色球）
- 系统自动识别当前轮到哪一方
- 显示当前最佳的进攻路线（纯色或花色，取决于轮到谁）
- 球进袋后自动更新桌面状态
- 一方打完自己所有球后打进黑8即获胜
- 比分自动记录和显示

### 路线推荐
- 系统分析桌面上所有可进攻的球
- 对每颗目标球评估进球难度
- 推荐最佳（最容易进球）的路线
- 显示：白球→目标球→球袋的完整路径

## 训练模式详细设计

### 10档难度等级

| 档位 | 难度 | 题型特点 |
|------|------|----------|
| 1 | 入门 | 白球、目标球、球袋在一条直线，直推 |
| 2 | 简单 | 小角度进球 |
| 3 | 初级 | 中等角度，需瞄准 |
| 4 | 中级 | 大角度，需精确瞄准 |
| 5 | 进阶级 | 需要低杆/高杆走位 |
| 6 | 挑战 | 需要加塞（侧旋） |
| 7 | 困难 | 需要翻袋（吃一库） |
| 8 | 高阶 | 两库翻袋 |
| 9 | 专家 | 组合球 + 复杂走位 |
| 10 | 大师 | 综合高级技巧 |

### 训练流程
1. 系统显示目标球和母球的指定位置
2. 选手手动摆球到指定位置
3. 系统通过摄像头验证摆球是否正确（误差允许范围内）
4. 系统给出推荐路线、杆法、力度、母球走位和落点
5. 选手击球
6. 系统判断：
   - 目标球是否进袋 ✓
   - 母球是否落在指定区域 ✓
   - 两者都满足 → 训练成功

### 闯关模式
- 每个关卡需要连续成功3次才能进入下一关
- 失败则重新计数
- 已通过的关卡可以手动选择复习

## 训练题库设计

每个难度档位至少包含5种不同的球形布局，总共至少50个训练题。

## 数据流

```
摄像头帧
  ↓
视觉识别模块 → 桌面状态（球的位置、颜色）
  ↓
物理引擎 → 最佳路线、杆法、力度
  ↓
┌────────────────────┬────────────────────┬───────────────────┐
↓                    ↓                    ↓                   ↓
投影渲染模块          API服务器            API服务器            API服务器
↓                    ↓                    ↓                   ↓
投影仪APP ← WiFi ── 手机APP ← WiFi ──── 比分网页 ← WiFi ── NAS/软路由
↓                                                            ↓
投影仪投射到桌面                                            液晶屏显示
```

### 安卓APP技术实现

#### 投影仪APP
- 使用 **Java** + Android SDK 开发
- **WebSocket客户端**：接收后端推送的JPEG图像，解码后全屏显示
- **自动重连**：与后端断开时每隔3秒尝试重连
- **心跳检测**：每10秒发送心跳包，保持连接
- **全屏沉浸模式**：隐藏状态栏和导航栏
- **最低支持 Android 5.0 (API 21)**（兼容大多数智能投影仪）

#### 手机APP
- 使用 **Java** + Android SDK 开发
- **网络服务发现**：通过UDP广播自动发现局域网内的后端电脑
- **WebSocket客户端**：接收实时桌面状态
- **REST客户端**：调用后端API进行控制操作
- **Material Design** 界面风格，适配手机屏幕

## WiFi摄像头建议

用户已有WiFi摄像头，需要确认：
- 支持RTSP协议
- 分辨率建议 1080P+
- 安装在台球桌正上方，俯拍
- 确保WiFi信号稳定（建议路由器和摄像头之间无遮挡）

## 附录

### 坐标系统
- 以台球桌面为二维坐标系
- 球袋坐标：四角各一个 + 两侧长边中间各一个 = 6个球袋
- 所有坐标标准化为 0.0 - 1.0 范围

### 物理参数
- 桌岸弹性系数：0.75-0.85
- 球与球摩擦系数：0.2
- 球与桌面摩擦系数：0.03
- 母球半径：标准台球尺寸
